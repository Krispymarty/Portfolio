# Interaction storyboard

## Narrative device

The same low-poly workstation evolves from dormant study space to an integrated AI-product system. Cyan traces learning and systems; orange traces projects, decisions, and output. The HTML story leads; the workstation confirms state.

## Arrival — initialize

- The title and essential actions are visible immediately.
- The workstation enters as a composed object rather than a separate demo.
- Lamp, monitor, and cyan path wake in sequence.
- Character transitions from idle_work to initialize and typing_loop.
- Primary action jumps to project proof; resume remains adjacent.

## Journey — acquire foundations

- The growth trace becomes the foreground composition.
- Workstation moves to a quiet persistent right-side state.
- Monitor shows a foundation-to-system map.
- Character alternates typing_loop and inspect_screen.
- Cyan pathways accumulate while orange remains restrained.

## Capabilities — trace relationships

- Capability filters remain direct buttons.
- Selecting Fraud, Vision, or LifeXP dispatches the same project context used by the monitor.
- Monitor switches to the chosen capability/project system.
- Character keeps typing with occasional short_break; no hover-only behavior.

## Projects — switch system

- Project tabs update accessible HTML, monitor content, orange pathways, module posture, and character project_switch/use_mouse states.
- Fraud shows evaluation/threshold language.
- Weapon Detection shows camera-to-alert inference flow.
- LifeXP shows current architecture with planned recommendation work separated.
- Exact repositories or live demos remain labelled as missing when not verified.

## Experience — integrate

- Cyan and orange pathways synchronize.
- Server modules and frame settle into a completed arrangement.
- Character checks the notebook, returns to work, and occasionally takes a short break.
- HTML proof remains visually dominant.

## Contact — resolve

- Character stops typing and uses contact_complete posture.
- Monitor displays the verified email and “open channel” state.
- Cyan and orange paths meet; environmental motion becomes still.
- Email, copy-email, LinkedIn, GitHub, resume, and return-to-origin remain conventional controls.

## Responsive behavior

- Desktop: persistent scene, controlled chapter repositioning, full project-coupled state.
- Tablet: balanced quality, reduced scene scale and shadow work, navigation menu always available when desktop links hide.
- Mobile: scene is an arrival-only lightweight composition; later chapters use typography and state labels instead of a fixed canvas.
- Reduced motion: static workstation schematic, no continuous canvas, no sticky journey motion, immediate content.

## Performance states

- High: full scene with restrained particles and one shadow source.
- Balanced: lower DPR, smaller shadow resources, no secondary particles.
- Lightweight: no real-time shadows/particles and simplified environment.
- Hidden tab: rendering pauses until visibility returns.

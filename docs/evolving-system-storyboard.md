# The Evolving System — storyboard

## Shared scene rules

One persistent procedural instrument occupies the spatial field. Its parts are a faceted evidence kernel, two measurement apertures, four system layers, chronological signal nodes, project modules, and a transmission beam. HTML remains the authoritative content layer. Camera movement is bounded and eased; pointer influence is under four degrees.

## Chapter 0 — Initialization / Dormant

- Camera: distant, slightly above the core; narrow field of view.
- Artifact: incomplete kernel, one dim aperture, disconnected nodes.
- HTML: concise calibration label only while Three.js genuinely initializes.
- Interaction: none required; skip link and page content are already available.
- Transition: the first successful render emits readiness and removes calibration.
- Lighting: low neutral rim with a small amber pilot signal.
- Mobile: no blocking overlay; static core appears immediately.
- Reduced motion: static dormant diagram, then immediate initialized state.

## Chapter 1 — Identity / Initialized

- Camera: balanced three-quarter view beside the headline.
- Artifact: kernel closes, primary aperture activates, four system plates align.
- HTML: name, positioning, introduction, proof, résumé, and work CTA.
- Interaction: bounded pointer inspection and layer buttons.
- Transition: scrolling grows the first pathway out of the kernel.
- Lighting: amber key light, faint violet explainability signal.
- Mobile: centered, shallow perspective, direct layer controls below.
- Reduced motion: no rotation; selected layer changes color and label instantly.

## Chapter 2 — Origin / Learning network

- Camera: moves laterally to reveal the instrument's chronological spine.
- Artifact: milestone nodes connect in order and remain lit once acquired.
- HTML: milestones alternate around the persistent axis with dates and verified detail.
- Interaction: scroll selects the nearest milestone; navigation can jump directly.
- Transition: the final journey node folds into the capability aperture.
- Lighting: node-by-node amber accumulation.
- Mobile: single reading rail, one visible active signal, no pinned travel.
- Reduced motion: all connectors displayed; active milestone uses outline and text.

## Chapter 3 — Capabilities / Capability system

- Camera: closer orthographic-feeling view for inspection.
- Artifact: capability satellites orbit four functional layers; selected evidence path brightens.
- HTML: category controls, technology lists, and project evidence links.
- Interaction: selecting All/Fraud/Vision/LifeXP changes connected nodes and project state.
- Transition: selected satellites detach and become project modules.
- Lighting: category-neutral chalk with selected amber/violet route.
- Mobile: satellites reduce to four nodes; buttons are the primary input.
- Reduced motion: relationship lines switch instantly without orbit.

## Chapter 4 — Projects / Project modules

- Camera: three controlled targets, one per project; no free orbit.
- Artifact: Fraud becomes a threshold aperture, Vision becomes a scanning gate, LifeXP becomes a layered product stack.
- HTML: distinct authentic diagram, controls, technical breakdown, outcomes, stack, and source links.
- Interaction: tabs, arrow keys, Home/End, instrument-state buttons, and direct project selection.
- Transition: leaving the chapter draws the selected modules back into the kernel.
- Lighting: fraud amber/violet, vision amber/green verification, LifeXP amber/chalk with planned elements dimmed.
- Mobile: all project articles stay readable; selected controls scroll into view; scene uses a single simplified module.
- Reduced motion: project change is an immediate state swap.

## Chapter 5 — Experience / Integrated system

- Camera: pulls back to show the complete instrument.
- Artifact: prior pathways reconnect, rings synchronize, and all acquired nodes remain visible.
- HTML: experience, education, and recorded achievements arranged as structural inputs rather than résumé cards.
- Interaction: focus and scroll highlight the relevant structural layer.
- Transition: synchronized rings open a channel toward contact.
- Lighting: fuller chalk fill and stable amber core.
- Mobile: compact proof sequence, simplified integrated glyph.
- Reduced motion: fully assembled static diagram.

## Chapter 6 — Transmission

- Camera: returns near the opening angle, now closer and complete.
- Artifact: front aperture opens and emits a restrained signal line; no particle explosion.
- HTML: email, LinkedIn, GitHub, résumé, role interests, and return-to-start link.
- Interaction: clear 44px actions; links do not depend on the scene.
- Transition: returning to the beginning restores the dormant/initialized framing and shows the progression loop.
- Lighting: quiet amber transmission with violet secondary trace.
- Mobile: one static open aperture above contact actions.
- Reduced motion: persistent beam and complete core with no animation.

## Rendering tiers

- High: full procedural geometry, smooth materials, all nodes, 1.5 DPR ceiling, dynamic camera and subtle pointer inspection.
- Balanced: fewer radial segments and nodes, 1.25 DPR ceiling, no expensive post-processing, event-driven rendering.
- Lightweight: CSS/static artifact, no WebGL canvas, direct controls and full HTML content preserved.

Visitors can manually select a tier; `prefers-reduced-motion`, data saving, constrained memory/CPU, and narrow mobile viewports bias the initial choice downward.

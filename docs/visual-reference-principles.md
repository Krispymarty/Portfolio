# Visual reference principles

References are used for principles only. No source code, exact layout, wording, asset, typography, color palette, avatar, camera path, or animation is copied.

## Bruno Simon portfolio

Reference: https://bruno-simon.com/

Useful principle: one navigable 3D world can make a developer’s technical medium immediately legible.

Adaptation: this portfolio keeps conventional scrolling and accessible HTML, while one low-poly workstation reacts to chapter and project state. The visitor never has to drive a vehicle or learn a game control.

Originality boundary: preserve Yashmit’s evidence, graphite/cyan/orange workstation, seated developer, and decision-system metaphor. Do not copy the environment, vehicle interaction, physics, copy, or world layout.

## Lusion Studio

Reference: https://www.lusionstudio.com/

Useful principle: WebGL can function as art direction when it is coordinated with typography, pacing, and transitions rather than treated as an isolated canvas.

Adaptation: the workstation shares state with the signal rail, capability filters, project tabs, monitor content, and contact resolution. Motion intensity drops outside signature moments.

Originality boundary: do not copy Lusion’s visual effects, studio presentation, layouts, shaders, or assets. The scene remains a personal computational studio built from local procedural geometry.

## Awwwards immersive homepage examples

Reference: https://www.awwwards.com/inspiration/immersive-homepage-gufram

Useful principle: a first viewport should establish one dominant visual idea and a clear invitation to continue.

Adaptation: the hero integrates identity, specific positioning, recruiter actions, and workstation initialization in one readable composition.

Originality boundary: awards-style pacing is not allowed to hide resume, projects, or contact. No preloader, custom cursor, audio gate, or forced cinematic sequence is introduced.

## Practical counter-reference

Reference: https://www.reddit.com/r/webdev/comments/1l28a6s/whats_the_best_portfolio_website_youve_ever_seen/

Useful principle: portfolio information must remain readable and immediately available; excessive animation is a common failure.

Adaptation: every 3D state has adjacent HTML evidence, persistent recruiter shortcuts, reduced-motion/static fallback, and direct anchors.

Originality boundary: the project does not reproduce any discussed site. The reference is used only as a usability constraint against awards-first excess.

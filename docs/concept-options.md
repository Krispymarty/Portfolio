# Creative Concept Options

## 1. Signal Observatory

- Central metaphor: a live observatory tracking the signals produced by Yashmit's projects and learning.
- Story progression: incoming signal → decoded experience → capability map → project constellations → live activity → contact beacon.
- Visual style: dark instrumentation, traces, precise plotting, sparse violet/amber signal color.
- 3D role: a node field whose focus shifts between evidence clusters.
- Interaction: visitors tune between chapters and inspect nodes.
- Strengths: fits GitHub/build activity and technical curiosity.
- Risks: easily collapses into a generic sci-fi dashboard or decorative particles.
- Performance: a WebGL node field would add meaningful bundle/GPU cost; CSS-only treatment would be safer but less expressive.

## 2. Model → Impact (selected)

- Central metaphor: a transparent decision engine that turns raw input into useful, explainable outcomes.
- Story progression: arrival/input → learning layers → capability stack → three decisions/projects → real-world experience → open channel.
- Visual style: optical-instrument black, chalk-white type, amber measurement, restrained inherited violet signal.
- 3D role: one layered CSS-3D Decision Core changes orientation and highlighted layer as the story moves from data to models to APIs and product impact.
- Interaction: scroll updates chapter progress; pointer movement offers restrained depth; selecting a project changes the core's active layer.
- Strengths: directly expresses the résumé's evolution from model performance to explainability and product delivery. It creates a coherent narrative without inventing a fictional world.
- Risks: system diagrams can become too literal or dense; copy and spacing must remain human.
- Performance: low. CSS transforms and a tiny observer/pointer client component avoid a 3D runtime and retain full content without JavaScript.

## 3. Systems Workbench

- Central metaphor: a digital lab bench where each project is a specimen with notes, tests, and measured outcomes.
- Story progression: desk arrival → field notebook timeline → tool wall → project specimens → credentials drawer → leave a note.
- Visual style: close-cropped artifacts, label plates, annotated screenshots, physical depth.
- 3D role: project artifacts tilt and separate into layers like exploded technical objects.
- Interaction: direct manipulation of specimens and expandable annotations.
- Strengths: tactile, personal, and naturally evidence-led.
- Risks: requires high-quality screenshots and artifacts that the repository currently lacks; synthetic substitutes would feel false.
- Performance: moderate due to large imagery and layered transforms.

## Recommendation

Model → Impact is the strongest fit. Yashmit's most credible story is not “futuristic developer”; it is the progression from training models, to explaining their decisions, to wrapping them in usable systems. The concept can be distinctive with current assets, scales cleanly to mobile, and gives 3D a specific explanatory purpose.

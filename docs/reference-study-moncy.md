# Reference study: moncy.dev

Studied on 17 July 2026 at desktop and mobile widths, with the public repository used only for high-level technical context. This is a principle study, not a reconstruction guide.

## 1. Sense of arrival

The reference opens with a restrained loading interlude, then resolves into a single, immediately legible hero composition. The central character is already the protagonist before the visitor reads anything. Large text, direct eye contact, and a stable top navigation establish identity quickly; small reactions during early scroll make the scene feel awake.

Adaptation: the portfolio should arrive as a quiet instrument powering on. Loading UI must reflect genuine scene readiness and disappear immediately when the enhanced scene is unavailable or already ready.

## 2. Relationship between 3D and HTML

The 3D subject and HTML are composed as one frame. HTML establishes identity and navigation while the 3D subject supplies continuity and emotional response. Later, the same visual world supports career, work, technology, and contact content rather than being replaced by unrelated hero decoration.

Adaptation: Yashmit's HTML remains semantic and complete, while one procedural computational instrument persists behind and beside it. The instrument explains data, model, explainability, and product integration; it never contains essential copy.

## 3. Scroll as an experience controller

Scroll changes pose, framing, scale, and surrounding content. It often advances a staged scene before releasing into the next chapter. The browser's native scroll remains the input, so the interaction is discoverable even when the visual response is ambitious.

Adaptation: section visibility and local progress update the instrument state, camera target, node activation, and lighting. No wheel interception, forced snapping, or hidden reading content.

## 4. Connected transitions

The reference preserves a dark spatial field, stable navigation, recurring lighting, and a persistent subject. Elements leave by transforming or moving out of the same composition that introduces the next element. This removes the sense of stacked landing-page blocks.

Adaptation: use a fixed scene, a continuous signal line, shared amber activation, and overlapping chapter thresholds. Typography and diagrams move through the same depth system while body copy remains stable.

## 5. Project introductions

Projects arrive as a distinct destination: a horizontally staged body of work with varied media rather than a uniform card grid. The work chapter changes pacing and makes screenshots part of the environment.

Adaptation: projects become selectable instrument modes. Fraud detection uses a decision-threshold plot, weapon detection uses a frame-processing path, and LifeXP uses a clearly labelled present/planned product system. Project controls, outcomes, stack, repository context, and keyboard navigation remain normal HTML.

## 6. Typography and motion

Large type acts as scene geometry. It frames the subject and creates rhythm; motion changes emphasis without making body copy chase the camera. Display movement is bolder than reading-text movement.

Adaptation: use the existing Manrope/JetBrains Mono system as measurement typography, with headlines aligned to instrument axes. Preserve the existing maximum scale and accessible line lengths. Motion uses assembly, pathway growth, and aperture changes rather than repeated fade-ups.

## 7. Personality through interaction

Small subject reactions, cursor response, visual state changes, and direct project controls make the reference feel authored. The interaction is playful without requiring game instructions.

Adaptation: the computational core can be inspected within safe rotation limits. Skill and project selections reconfigure real semantic relationships. Pointer response remains subtle; touch uses direct controls.

## 8. Continuity

Continuity comes from the repeated visual protagonist, a stable spatial palette, fixed navigation, and section changes that modify the existing world instead of replacing it. The closing scene still belongs to the opening scene.

Adaptation: the initial incomplete instrument returns at contact as the same object, now integrated and transmitting. The chapter rail exposes position and provides escape hatches throughout.

## 9. Principles safe to adapt

- One persistent visual protagonist across the full narrative.
- Native scroll as a state signal for camera, lighting, and structure.
- HTML and 3D composed together but independently understandable.
- Project-specific visual behaviour rather than identical cards.
- A stable navigation and progress model across experimental scenes.
- Purposeful first-load choreography tied to real readiness.
- Mobile art direction that reduces depth and transitions instead of shrinking desktop.
- Reduced-motion and non-WebGL experiences with the same content and controls.

## 10. Elements We Must Not Copy

- The custom avatar, its silhouette, poses, facial reactions, clothing, materials, or animation.
- The room, workstation, seated-character scene, desk, monitor, or career tableau.
- The black/purple/pink palette and rim-light treatment.
- The exact hero layout, navigation placement, social rail, type treatment, or camera path.
- The exact loading capsule, project grid, horizontal composition, floating technology balls, or glowing cursor orb.
- Source components, animation values, custom assets, trial GSAP plugins, or repository structure.
- Any asset extracted from the live site. The repository explicitly states that the avatar is proprietary and that the complete design must not be replicated.

## High-level technical context

The public repository identifies React, TypeScript, GSAP, Three.js, and WebGL. It also states that trial-only GSAP plugins are unsuitable for production and asks developers not to clone the full site or design. This implementation therefore uses the project's existing React/Next.js/Three.js stack, native observers and scroll signals, and original procedural geometry only.

## Conclusion

The benchmark is not the reference's character or scene. It is the coherence created when one visual subject, one motion language, and one scroll narrative persist long enough to become a world. For Yashmit, that world is an evolving measurement instrument: precise, evidence-led, and visibly more capable at the end than at the beginning.

# Creative iteration audit

## Current strengths

- Trustworthy, resume-backed content is centralized in `src/data/portfolio.ts`.
- Semantic sections, a skip link, keyboard project tabs, focus treatment, and reduced-motion rules already exist.
- The Decision Core, chapter rail, journey sequence, capability filtering, and project instruments establish a strong technical foundation.
- The black/chalk/amber/violet optical-lab identity is original and appropriately distinct from the reference.

## Gaps against the new benchmark

### Continuity

The fixed signal scene is present, but its four flat planes read as a decorative hero object. Most chapter transitions are still perceived as separate dark sections because the artifact's silhouette and function do not change enough.

### Narrative evolution

Chapter events currently adjust rotation, layer spacing, and opacity. They do not yet communicate dormant, initialized, learning, capability, project, integrated, and transmission states as recognizably different stages of one system.

### Camera and lighting

The existing scene uses one camera position and unlit basic materials. This limits storytelling and makes state changes feel like CSS transforms instead of camera-directed spatial moments.

### Projects

The project lab is substantially better than a card grid and already uses authentic diagrams. It needs a stronger visual connection to the persistent artifact so project selection visibly reconfigures the system rather than only updating a separate panel.

### Mobile

Mobile content is complete, but the enhanced scene needs an explicit lightweight tier, fewer nodes, constrained camera travel, touch-safe direct controls, and no reliance on pointer response.

### Loading

The calibration overlay is time-based and appears even before it knows whether assets need loading. It should be tied to the scene's real ready/unavailable event and should never delay the static experience.

### Content quality

Several source files contain mojibake in dashes, quotes, “Résumé,” and metric symbols. These errors reduce trust and must be repaired.

## Opportunities

- Replace the plane stack with an original modular core: kernel, aperture rings, chronological node paths, capability satellites, and three project modules.
- Introduce controlled camera targets and palette-light shifts per chapter without moving body copy.
- Use one event contract for chapter, journey, capability, project, pointer, visibility, and readiness state.
- Expose High, Balanced, and Lightweight rendering tiers through automatic detection and an accessible quality control.
- Make the integrated and transmission states visually echo the dormant opening state.
- Keep the CSS fallback as a deliberate diagram rather than an apology for missing WebGL.

## Success criteria

The redesign succeeds when a visitor can recognize the same instrument in every chapter, name how it changed, move between projects with keyboard or pointer, understand all claims without the canvas, and reach contact or résumé immediately. It must remain clearly unlike moncy.dev in subject, palette, composition, assets, and movement.

# The Living Developer Workspace — storyboard

## Experience model

The workstation is one persistent computational instrument with two equivalent routes through the same evidence.

- Guided Story Mode is the default. Scrolling controls chapter state and all content remains normal semantic HTML.
- Explore Mode is optional. It permits bounded orbit on desktop and direct object selection at every size. It never unlocks unique essential information.
- Skip Experience leads directly to the project case studies.

## Entry: 0.0–2.6 seconds

1. The procedural scene mounts dormant: the screen, lamp, and paths have zero or near-zero emission.
2. The startup headline reveals in two masked lines with an 80ms stagger.
3. The lamp and monitor ease toward the initialization state.
4. The seated developer moves from rest toward an attentive working posture.
5. Cyan system pathways begin carrying the first signal.
6. At 2.2 seconds the copy exits and the portrait-led identity becomes visible.
7. By 2.6 seconds the scene, name, role, actions, navigation, and exploration control are settled.

The intro is skippable by its control, pointer, keyboard, touch, or wheel. Reduced motion and no JavaScript expose identity immediately.

## Guided chapters

### 01 — Home / Initialize

The portrait and verified role lead. The workstation is fully legible but remains secondary to identity and recruiter actions.

### 02 — About / Journey

Camera depth relaxes, cyan is dominant, and the character alternates between typing and checking the notebook. The growth trace explains the move from foundations to applied systems.

### 03 — Skills / Capability graph

The rear frame and cyan relationships activate. HTML capability groups provide the complete readable equivalent.

### 04 — Projects / Monitor focus

Orange becomes more active. Project selection updates the monitor texture, project module angle, character attention, and rear-system state. Each case study uses its own instrument view and retains complete semantic details.

### 05 — Experience / Integration

Platform and server modules settle into a more integrated configuration. Education, work, leadership, and recorded signals share one evidence chapter.

### 06 — Contact / Transmit

The scene resolves rather than escalating: both pathways align, the lamp brightens, and the character sits back. Email, LinkedIn, GitHub, and résumé remain direct HTML actions.

## Explore Mode

Entering Explore Mode raises the workstation to an interaction layer without moving the visitor to another page. A visible object selector, inspector panel, Return to Guided Mode control, and Skip Experience link are always present.

- Monitor: projects and project-to-scene switching.
- Server: APIs, data, deployment, and product engineering.
- Notebook: education, foundations, and current learning.
- Computational frame: capability categories tied to evidence.
- Lamp: concise current-focus material.

Desktop supports damped, bounded orbit with no pan or zoom. Mobile uses predefined composition and touch selection only.

## Motion grammar

- Identity startup: masked vertical reveal and restrained blur, once per page load.
- Journey: paced trace movement and notebook-aware character state.
- Capabilities: node activation and relationship emphasis.
- Projects: selection, monitor content change, and orange path response.
- Experience: weighted module settlement.
- Contact: light and path convergence.

The system uses opacity, transforms, camera framing, material emission, and damped object state. It does not animate layout properties or apply one repeated fade-up treatment to every chapter.
